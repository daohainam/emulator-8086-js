Registers, Flags & Memory

The CPU's inner world — its scratchpad, mood ring, and vast storage vault

The CPU's scratchpad: 14 registers

Think of registers like the sticky notes on a chef's workstation — a tiny handful of labeled spots where the cook keeps the numbers they're actively working with right now. The 8086 has 14 of them:

General Purpose (Arithmetic)

AX Accumulator
BX Base
CX Counter
DX Data

Each is 16 bits wide. AX can be split into AH (high byte) and AL (low byte).

Index & Pointer

SI Source Index
DI Dest Index
SP Stack Pointer
BP Base Pointer

Used for pointing to locations in memory. SP tracks the top of the stack. IP (not shown) holds the next instruction address.

Segment Registers

CS Code Segment
DS Data Segment
SS Stack Segment
ES Extra Segment

These point to 64 KB "windows" into the 1 MB address space. CS:IP together point to the next instruction.

What registers look like in the emulator code

All CPU state lives in a single JavaScript object. Registers are just properties of that object — nothing magical:

CPU8086.js 

const getReg16 = (idx) =>
  e.reg[
    ["AX","CX","DX","BX",
     "SP","BP","SI","DI"][idx]
  ];
const setReg16 = (idx, val) =>
  e.reg[["AX","CX","DX","BX",
         "SP","BP","SI","DI"][idx]]
    = val & 0xFFFF;
PLAIN ENGLISH

Define a function that reads a 16-bit register by its numeric index...

Look up the register name in an ordered list (AX=0, CX=1, DX=2...) ...

...the 8086 hardware defines this exact order — it matches the CPU encoding.

Define a function that writes a value to a 16-bit register...

...same name lookup using the index number...

...but mask to 16 bits (0xFFFF) so it never stores more than 65535. This is how real hardware works.

Flags — the CPU's mood indicators

After every calculation, the CPU automatically updates a set of flags. Think of them like dashboard warning lights on a car — they tell you what just happened:

ZF

Zero Flag

Set to 1 when the result of an operation is zero. Powers instructions like JZ (jump if zero).

CF

Carry Flag

Set when an addition overflows or a subtraction underflows. Essential for multi-byte arithmetic and detecting errors.

SF

Sign Flag

Mirrors the top bit of the result. If SF=1, the result is negative (in signed arithmetic). Powers JL / JG comparisons.

DF

Direction Flag

Controls whether string operations move forward (DF=0) or backward (DF=1) in memory. Set with STD / cleared with CLD.

IF

Interrupt Flag

When IF=1, hardware interrupts (keyboard, timer) are allowed. When IF=0, the CPU ignores them — like "do not disturb" mode.

The 1 MB memory vault — accessed via zip codes

The 8086 has a 1 MB address space. But the registers are only 16-bit, which can only address 64 KB. How do you reach all 1 MB? With a clever trick called segmented addressing.

Think of memory like a city with ZIP codes. The segment register is the ZIP code (neighborhood). The offset register is the street address. Together they pinpoint any house in the city:

CPU8086.js 

// Total address space (1 MB)
const ADDR_SPACE = 1048576;

// Convert segment:offset to physical address
const calcPhys = (s, o) =>
  (((s & 0xFFFF) << 4)
  + (o & 0xFFFF))
  & (ADDR_SPACE - 1);
PLAIN ENGLISH

The entire memory is 1,048,576 bytes (1 MB) — defined as a constant.

A function that converts a segment + offset pair to a single physical address...

Take the segment value and shift it left by 4 bits (multiply by 16). This gives a 20-bit base address.

Add the offset value to it...

...and wrap around if it exceeds 1 MB (the & mask keeps it in range). Result: any byte in the 1 MB vault is reachable.

🗺️
The formula: physical = (segment × 16) + offset

Example: CS=0x0000, IP=0x0100 → physical address = (0 × 16) + 256 = 256. That's where your program's first instruction lives when you use ORG 100h. You can verify this in the Memory Viewer!

The 1 MB memory map — neighborhoods with reserved addresses

Not all of memory is equally free. Certain address ranges are reserved for hardware — like land zoned for government buildings:

0x00000–0x003FF
Interrupt Vector Table — 256 pointers to interrupt handlers (4 bytes each)
0x00400–0x004FF
BIOS Data Area (BDA) — keyboard buffer, cursor position, hardware status
0x07C00
Boot Sector Load Point — BIOS copies disk sector 0 here, then jumps to it
0xB8000–0xBBFFF
VGA Text VRAM — the 80×25 screen. Write here → characters appear on the monitor
0xFFFFF
Reset Vector — the very first address the CPU executes after power-on

How the emulator packs 8 flags into one 16-bit number

All flags are stored in a single 16-bit flags register. The emulator uses two functions — packFlags and unpackFlags — to convert between the object form and the raw hardware integer:

CPU8086.js 

const packFlags = (e) => {
  const f = e.flags; let r = 0;
  if (f.CF) r |= (1<<0);
  if (f.ZF) r |= (1<<6);
  if (f.SF) r |= (1<<7);
  if (f.IF) r |= (1<<9);
  if (f.DF) r |= (1<<10);
  if (f.OF) r |= (1<<11);
  return r;
};
PLAIN ENGLISH

A function to pack all flags into one 16-bit hardware number...

Start with result = 0 (all bits off)...

If Carry Flag is set, turn on bit 0 of the result (hardware position for CF).

If Zero Flag is set, turn on bit 6 (the 8086 hardware specification says ZF lives at bit 6).

Sign Flag → bit 7. Interrupt Flag → bit 9. These positions are fixed by Intel's specification.

Direction Flag → bit 10. Overflow Flag → bit 11.

Return the single packed number — this is the exact format real 8086 hardware uses in the PUSHF instruction.

How a simple instruction changes everything

Let's trace what happens when the CPU executes CMP AX, 0 — the instruction asks "is AX equal to zero?"

IP
Instruction Pointer (CS:IP)

Fetching next instruction from memory at 0x0100... it says CMP AX, 0

DE
Binary Decoder

I decoded byte 0x3D from memory. That means CMP AX, immediate. Fetching the 16-bit value to compare against...

AX
AX Register

My current value is 0x0000 (zero). The comparison is: 0 - 0 = 0.

FL
Flags Register

Result is zero! I'm setting ZF=1, SF=0, CF=0, OF=0. Now any JZ instruction will jump!

IP
Instruction Pointer

IP advances past CMP to the next instruction. Ready for whatever comes next — JZ, JNZ, or just the next statement.
